\section{数据编译与训练命令}

中等任务扩散数据编译命令如下：

\begin{verbatim}
uv run --extra cuda -m src.policy.diffusion.compile \
    --config config/dp_medium.yaml
\end{verbatim}

训练命令如下：

\begin{verbatim}
uv run --extra cuda -m src.policy.diffusion.train \
    --config config/dp_medium.yaml
\end{verbatim}

如需从最近检查点恢复训练，可使用：

\begin{verbatim}
uv run --extra cuda -m src.policy.diffusion.train \
    --config config/dp_medium.yaml \
    --resume checkpoints/diffusion_medium/last.pt
\end{verbatim}

\section{评估与指标命令}

本文定稿评估以\path{config/thesis_eval}下的环境矩阵为准。为避免评估进程之间竞争GPU、物理仿真或日志输出资源，本地复现实验一次只启动一个评估命令；单个命令内部可使用6个worker。扩散策略在真值完整中等环境上的评估命令如下：

\begin{verbatim}
python -m src.main eval \
    --env-config config/thesis_eval/gt_medium/full.yaml \
    --policy-config config/dp_medium.yaml \
    --episodes 200 \
    --workers 6 \
    --param device=xpu \
    --log-file logs/thesis_eval_gt_full_dp_medium_n200.jsonl
\end{verbatim}

含噪完整中等环境可替换环境配置运行：

\begin{verbatim}
python -m src.main eval \
    --env-config config/thesis_eval/noisy_medium/full_moderate.yaml \
    --policy-config config/dp_medium.yaml \
    --episodes 200 \
    --workers 6 \
    --param device=xpu \
    --log-file logs/thesis_eval_noisy_full_moderate_dp_medium_n200.jsonl
\end{verbatim}

若需要复现MPC专家基线，可将\texttt{--policy-config}替换为\path{config/policy_expert.yaml}。批量矩阵评估需要逐个遍历以下两组环境配置，且一次只运行一个评估命令：

\begin{itemize}
    \item 真值矩阵：\path{config/thesis_eval/gt_medium}
    \item 含噪矩阵：\path{config/thesis_eval/noisy_medium}
    \item 扩散策略：\path{config/dp_medium.yaml}
    \item MPC专家：\path{config/policy_expert.yaml}
\end{itemize}

评估结束后，使用以下命令从JSONL日志中生成成功率、置信区间、最终距离、跟踪误差、平滑度、完成速度和吞吐量报告：

\begin{verbatim}
python -m src.eval.report \
    logs/thesis_eval_gt_full_dp_medium_n200.jsonl
\end{verbatim}

本文用于补充过程指标的本地矩阵命令如下：

\begin{verbatim}
source .venv/bin/activate
bash scripts/run_thesis_eval_matrix.sh

python -m src.eval.report \
    logs/thesis_eval_20260505_n200_dp_medium_xpu_w6/
\end{verbatim}

该脚本默认使用\texttt{EPISODES=200}、\texttt{WORKERS=6}、\texttt{DEVICE=xpu}和\path{config/dp_medium.yaml}，并把日志写入\path{logs/thesis_eval_20260505_n200_dp_medium_xpu_w6}。定稿前建议把每次运行的JSONL日志、命令行参数、git提交号和硬件信息一起归档。
